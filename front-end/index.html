Три кроки до прогресу

step 01

Додай тему

Створи тему яку хочеш вивчити та встанови пріоритет — від P1 (критично) до P5 (колись).

P1 React Hooks
P2 Docker Compose
P3 SQL joins
P4 WebSockets
step 02

Фіксуй що зробив

Щодня записуй прогрес по темі — що вивчив, написав, розібрав. Повна хронологія як git log.

2024-01-15 React Hooks
Вивчив useState, useEffect.
Написав 3 компоненти.
2024-01-16 React Hooks
Розібрав useContext...
step 03

Не переривай серію

Кожен день з активністю збільшує твій streak. Dashboard показує поточну серію і рекорд.

🔥 14 // поточний streak
рекорд: 21 день
// можливості

Все що потрібно для росту

01 // topics

Трекер тем

Організуй все що хочеш вивчити в одному місці. Від ідеї до завершення.

  • Пріоритети P1–P5 (від критичного до "колись")
  • Статуси: не почато → в процесі → завершено
  • Фільтрація по статусу
  • Редагування і видалення тем
02 // progress

Журнал прогресу

По кожній темі — хронологічний лог того що ти зробив. Як git commits, але для навчання.

  • Нотатки прив'язані до конкретної теми
  • Автоматична дата запису
  • Повна хронологія по кожній темі
  • Будь-який текст: посилання, нотатки, висновки
03 // streak

Streak система

Мотивуй себе через консистентність. Кожен запис прогресу — день в серії.

  • Поточний streak у реальному часі
  • Рекордна серія за весь час
  • Автоматичний скид якщо день пропущено
  • Відображення на головній сторінці
04 // stats

Статистика

Повна картина твого прогресу: скільки зроблено, де найбільше активності, як росте streak.

  • Розподіл тем за статусом (графік)
  • Розподіл за пріоритетами
  • Найактивніша тема
  • Загальна кількість записів прогресу
05 // dashboard

Dashboard

Головна сторінка — швидкий огляд усього що відбувається. Один екран замість п'яти.

  • Всі ключові цифри одразу
  • Останні додані теми
  • Швидке додавання нової теми
  • Поточний і рекордний streak
06 // auth

Авторизація

Твої дані — тільки твої. Кожен користувач бачить лише свої теми і прогрес.

  • Реєстрація та логін через email
  • JWT-токени для захисту API
  • Ізольовані дані кожного юзера
  • Автоматичний редирект якщо не авторизований
